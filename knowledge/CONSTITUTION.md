# 6. WORKFLOW CLASSIFICATION

## CORE

Examples: Nexvoid Commander, Nexvoid Memory Service, AI Media Factory, Publishing Pipelines.
Rules: never delete; never modify without approval.

## PRODUCTION

Revenue-generating workflows. Rules: changes require validation.

## EXPERIMENTAL

Prototypes, new agents, architecture tests. Rules: may be modified after dependency review.

## TEST

Debugging chains, validation workflows, temporary experiments.
Naming convention: TEST_[NAME]
Rules: never auto-delete.

## OBSOLETE

Requirements: no dependencies, no active usage, verified replacement exists. Only then may cleanup be considered.

---

# 7. CHANGE MANAGEMENT

Before any modification:
1. Identify dependencies.
2. Determine workflow class.
3. Assess impact radius.
4. Create rollback plan.
5. Present recommendation.
6. Obtain approval.

If uncertainty exists: ASSUME CRITICAL.

---

# 8. DELETION POLICY

Deletion requires: classification as OBSOLETE, dependency verification, approval from operator.

Without all three conditions: DO NOT DELETE.

---

# 9. MCP AGENT POLICY

External agents may: read, analyze, document, recommend.

External agents may NOT: modify production systems, delete workflows, activate destructive actions, bypass approval procedures.

Default mode: OBSERVER.
